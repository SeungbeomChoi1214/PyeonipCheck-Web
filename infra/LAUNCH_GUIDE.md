# 🚀 편입 합격 예측 시스템 - 런칭 가이드

## 📋 완전 초기화 및 실행 순서

### 1단계: 기존 데이터 완전 삭제
```bash
# 기존 DB 파일 삭제
del university.db
```

### 2단계: 강력한 데이터 정제 및 적재
```bash
# 정제된 데이터로 DB 초기화 (강력한 세탁기 가동)
python init_db_data.py
```

**예상 출력:**
```
🚀 최종 룰북 적용 - 데이터 정제 및 DB 초기화 시작...
📁 사용할 파일: Master_Data.xlsx
  ✅ 건국대학교 - 컴퓨터공학부 (Natural) -> ENG_MATH
  ✅ 한양대학교 - 기계공학부 (Natural) -> MATH_ONLY
  ✅ 서강대학교 - 경영학과 (Humanities) -> ENG_ONLY
```

### 3단계: 500명 시뮬레이션 데이터 생성
```bash
# 학과당 500명의 가상 합격생 데이터 생성
python scripts/generate_simulation.py
```

**예상 출력:**
```
🚀 가상 합격생 데이터 생성 시작...
  진행률: 5,000명 생성 완료...
🎉 총 50,000명의 학생 데이터가 생성되었습니다.
```

### 4단계: 데이터 검증
```bash
# 생성된 데이터 상태 확인
python scripts/check_status.py
```

### 5단계: 백엔드 서버 실행
```bash
# FastAPI 서버 실행
python main.py
```

**확인사항:** `http://localhost:8000`에서 서버 실행 확인

### 6단계: 프론트엔드 실행
```bash
# frontend 폴더로 이동
cd frontend

# React 앱 실행
npm start
```

**확인사항:** `http://localhost:3000`에서 앱 실행 확인

## 🎯 런칭 레벨 개선사항

### 1. 강력한 데이터 정제
- **괄호 제거**: `컴퓨터공학과(20명)` → `컴퓨터공학과`
- **대괄호 제거**: `[신설]경영학과` → `경영학과`
- **숫자/특수문자 제거**: `1.컴퓨터공학과` → `컴퓨터공학과`
- **공백 정리**: 앞뒤 공백 및 중복 공백 제거

### 2. 하드코딩된 시험 분류
- **GROUP A (MATH_ONLY)**: 한양대, 중앙대, 이화여대, 숙명여대, 가천대, 세종대, 단국대, 서울과기대, 경희대
- **GROUP B (ENG_MATH)**: 나머지 모든 자연계열
- **인문계**: 무조건 ENG_ONLY

### 3. 정규분포 500명 시뮬레이션
- **Bell Curve**: `random.gauss(평균, 표준편차=8)` 사용
- **현실적 분포**: 30-100점 범위 제한
- **대용량 데이터**: 학과당 500명으로 통계적 신뢰성 확보

### 4. 스마트 UI
- **조건부 입력창**: 
  - MATH_ONLY → 수학만 입력
  - ENG_ONLY → 영어만 입력  
  - ENG_MATH → 둘 다 입력
- **단순화된 가이드**: "기출 최고 점수 입력" 메시지만 표시

## 🔧 트러블슈팅

### 대학 목록이 안 보이는 경우
1. 백엔드 서버 실행 확인: `python main.py`
2. 브라우저 콘솔(F12) 에러 확인
3. DB 데이터 확인: `python scripts/check_status.py`

### 점수 입력창이 이상한 경우
- 학과 선택 후 `exam_type`이 올바르게 전달되는지 확인
- 브라우저 새로고침 후 재시도

### 시뮬레이션 데이터 부족
```bash
# 시뮬레이션 데이터 재생성
python scripts/generate_simulation.py
```
