# 🚀 편입 합격 예측 시스템 - 최종 실행 가이드

## 📋 실행 순서

### 1단계: 기존 DB 삭제 및 정제된 데이터 적재

```bash
# 1. 기존 DB 파일 삭제 (있다면)
del university.db

# 2. 정제된 데이터로 DB 초기화
python scripts/init_db_data.py

# 3. 시뮬레이션 데이터 생성 (가상 합격생 100명씩)
python scripts/generate_simulation.py

# 4. 데이터 상태 확인
python scripts/check_status.py
```

### 2단계: 백엔드 서버 실행

```bash
# FastAPI 서버 실행
python main.py
```

서버가 `http://localhost:8000`에서 실행됩니다.

### 3단계: 프론트엔드 실행

```bash
# frontend 폴더로 이동
cd frontend

# React 앱 실행
npm start
```

브라우저에서 `http://localhost:3000`으로 접속합니다.

## 🛡️ 주요 고도화 기능

### 1. 데이터 정제 (`init_db_data.py`)
- **정규식 적용**: 괄호, 숫자, 특수문자 제거
- **중복 방지**: get_or_create 패턴으로 안전한 데이터 적재
- **자동 분류**: 학과명 기반 인문/자연계 자동 판별

### 2. 허수 방어 시스템 (`main.py`)
- **스팸 탐지**: 98점 초과, 합계 195점 초과 등 비현실적 점수 탐지
- **쉐도우 밴**: 허수는 1등으로 표시하되 실제 랭킹에서는 제외
- **데이터 분리**: `is_spam=False` 데이터만 통계 산출에 사용

### 3. UI/UX 개선
- **입력 가이드**: "Career High 점수 입력" 안내
- **연도 선택**: 2025/2024/기타 라디오 버튼
- **추가 기능**: "도전 가능한 대학 라인업" 버튼 (준비중)
- **허수 경고**: 비현실적 점수 입력 시 경고 메시지

## 📊 데이터 흐름

1. **원본 데이터** → **정제** → **DB 저장**
2. **사용자 입력** → **허수 탐지** → **랭킹 계산**
3. **정상 데이터만** → **백분위 산출** → **진단 결과**
